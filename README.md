# ☁️ AustriaWeather — Azure Data Engineering Project

A fully serverless, cloud-native ETL pipeline built on Microsoft Azure that collects, transforms, stores, and visualizes real-time weather data for Austrian cities — deployed entirely via the Azure Portal and Azure Cloud Shell at zero cost.

---

## 🏗️ Architecture

```
OpenMeteo API (free, public)
        │
        ▼
⚡ Azure Functions (Timer Trigger — hourly)
        │
        ├──▶ 🗄️ Blob Storage
        │         ├── bronze/   (raw JSON)
        │         ├── silver/   (transformed)
        │         └── gold/     (aggregated)
        │
        └──▶ 🌍 Cosmos DB (NoSQL)
                  │
                  ▼
        ⚡ Azure Functions (HTTP Trigger — REST API)
                  │
                  ▼
        🌐 Static Web App (Chart.js Dashboard)
```


## ☁️ Azure Components

### ⚡ Azure Functions
- **Plan:** Consumption (Serverless)
- **Runtime:** Python 3.12
- **Functions:**
  - `fetch_weather` — Timer Trigger, runs hourly, fetches weather data from OpenMeteo API for Graz, Wien and Salzburg, writes raw data to Blob Storage (Bronze), transforms to Silver and loads into Cosmos DB
  - `get_weather_api` — HTTP Trigger (ANONYMOUS), REST API endpoint that queries Cosmos DB and returns JSON for the dashboard

### 🗄️ Azure Blob Storage — Medallion Architecture
- **Redundancy:** LRS (Locally Redundant Storage)
- **Containers:**
  - `bronze` — raw API responses as JSON, partitioned by `year/month/day/hour`
  - `silver` — normalized and cleaned data (standardized units, filtered fields)
  - `gold` — aggregated daily summaries (min/max/avg temperature per city)

### 🌍 Azure Cosmos DB
- **API:** NoSQL
- **Database:** `WeatherDB`
- **Container:** `Measurements` — partition key `/city`
- Stores transformed Silver-layer records and Gold-layer daily aggregates
- Queried via REST API using parameterized SQL

### 🌐 Azure Static Web Apps
- Hosts a single-page dashboard built with HTML, CSS and Chart.js
- Displays hourly temperature trends per city via dropdown selection
- Connected to GitHub repository for automatic deployment

### 🔄 CI/CD — GitHub Actions
- Workflow automatically generated by Azure Static Web Apps on connection
- Every `git push` to `main` triggers a new deployment
- Zero manual deployment steps required for frontend changes

### 📁 Azure Resource Group
- All project resources are grouped for easy management and cleanup

## 🚀 Key Learnings

- Designing and deploying a **serverless ETL pipeline** on Azure from scratch
- Implementing the **Medallion Architecture** (Bronze / Silver / Gold) on cloud storage
- Working with **Azure Functions** Python programming model (v2)
- Managing **secrets and connection strings** via Application Settings (Environment Variables)
- Configuring **CORS** for cross-origin API access between Static Web App and Functions
- Setting up **CI/CD** with GitHub Actions for automated frontend deployment
